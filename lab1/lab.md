# Анализ предметной области

## Введение

**Тема:** Информационная система учета производства и кадров авиастроительного предприятия

**Название:** AeroProduction ERP

**Планируемые внешние интеграции:** 1С:Предприятие, системы учета кадров, системы управления качеством, САПР-системы

**Описание:** Система представляет собой комплексное решение для управления производственными процессами авиастроительного предприятия. Она обеспечивает учет организационной структуры (цеха, участки), выпускаемой продукции (самолеты, вертолеты, ракеты и др.), кадрового состава (ИТР и рабочие), производственных циклов, а также процессов испытаний готовой продукции. В отличие от универсальных ERP-систем, решение учитывает специфику авиастроительной отрасли с её сложной иерархией персонала, многоэтапными циклами сборки и обязательными испытаниями продукции.

---

## Предметная область

### Целевая аудитория

Целевой аудиторией системы являются:

- **Руководство предприятия** — для контроля производственных показателей, кадрового состава и выполнения плановых заданий
- **Начальники цехов и участков** — для управления подчиненными подразделениями, распределения работ и контроля выполнения
- **Мастера** — для оперативного управления бригадами и контроля выполнения конкретных операций
- **Кадровая служба** — для учета движения кадров, формирования отчетности по персоналу
- **Планово-производственный отдел** — для планирования производственных циклов и загрузки мощностей
- **Служба качества** — для контроля прохождения изделиями испытаний и анализа результатов
- **Бухгалтерия** — для учета выпущенной продукции и начисления заработной платы

## Хранимые сущности

### Сущности и атрибуты

| Сущность | Атрибуты |
|----------|----------|
| **Предприятие** | Название, адрес, директор |
| **Цех** | Номер, название, специализация, начальник цеха |
| **Участок** | Номер, название, цех, начальник участка |
| **Категория изделия** | Название (самолет, вертолет, ракета, планер, дельтаплан, прочие) |
| **Изделие** | Наименование, категория, обозначение, дата начала сборки, дата окончания сборки, цех сборки |
| **Самолет** | Изделие, тип (гражданский, транспортный, военный), число двигателей, дальность полета, вместимость |
| **Ракета** | Изделие, тип (артиллерийская, авиационная, военно-морская), мощность заряда, дальность поражения |
| **Вертолет** | Изделие, число винтов, грузоподъемность, максимальная высота полета |
| **Категория персонала** | Название (инженер, технолог, техник, сборщик, токарь, слесарь, сварщик и др.) |
| **Персонал** | ФИО, дата рождения, дата приема, дата увольнения, категория, разряд/квалификация |
| **Бригада** | Номер, название, участок, бригадир, дата формирования |
| **Мастер** | Персонал (ИТР), участок, дата назначения |
| **Цикл работ** | Наименование, описание, нормативная длительность |
| **Производственный процесс** | Изделие, участок, цикл работ, бригада, дата начала, дата окончания, статус |
| **Испытательная лаборатория** | Название, тип (полигон, лабораторный стенд), адрес |
| **Оборудование** | Инвентарный номер, название, тип, лаборатория, дата ввода в эксплуатацию |
| **Испытатель** | Персонал, специализация, допуски |
| **Испытание** | Изделие, лаборатория, дата начала, дата окончания, результат, протокол |

### Особенности

- Каждая категория изделий имеет специфические атрибуты (наследование через отдельные таблицы)
- Бригадиры выбираются из числа рабочих
- Мастера, начальники участков и цехов назначаются из ИТР
- Одно изделие проходит последовательность циклов работ на разных участках
- Одна бригада выполняет все работы по сборке изделия на конкретном участке
- На одном участке может работать несколько бригад одновременно
- Различные изделия могут проходить одинаковые циклы работ
- Испытательная лаборатория может обслуживать несколько цехов
- Один цех может пользоваться несколькими лабораториями
- При испытании изделия может использоваться различное оборудование
- Ведется история движения кадров (прием, увольнение, перемещение)

## Проблема и текущее состояние

### Конкуренты

Существующие решения на рынке представлены как универсальными ERP-системами, так и специализированными решениями для машиностроения.

#### **SAP ERP**

Источник: https://www.sap.com

Универсальная ERP-система с модулями для производственного предприятия.

- **Сильные стороны:** полнофункциональная система, глубокая интеграция всех модулей, масштабируемость, поддержка международных стандартов.
- **Слабые стороны:** высокая стоимость внедрения и лицензий, сложность настройки под специфику авиастроения, избыточная функциональность, длительные сроки внедрения.

#### **1С:ERP Управление предприятием**

Источник: https://v8.1c.ru

Российская ERP-система для производственных предприятий.

- **Сильные стороны:** относительно доступная стоимость, широкая распространенность в России, наличие сертифицированных специалистов, интеграция с бухгалтерией.
- **Слабые стороны:** недостаточная гибкость для учета специфики авиастроения (сложная иерархия персонала, многоэтапные испытания), ограниченные возможности мобильного доступа.

#### **КОМПАС-Автоматизация управления**

Источник: https://ascon.ru

Решение для машиностроительных предприятий с интеграцией САПР.

- **Сильные стороны:** учет специфики машиностроения, интеграция с конструкторской документацией, российская разработка.
- **Слабые стороны:** ограниченные возможности учета кадров, слабая поддержка сложных производственных циклов с перемещением между участками, недостаточная функциональность модуля испытаний.

#### **Галактика ERP**

Источник: https://galaktika.ru

Отечественная ERP-система для крупных производственных предприятий.

- **Сильные стороны:** модульная архитектура, поддержка холдинговых структур, российская разработка с учетом местного законодательства.
- **Слабые стороны:** отсутствие специализированного функционала для авиастроения, высокая стоимость, сложность доработки под специфические требования.

### Существующие проблемы и недостатки

Основные требования к искомой системе следующие:

- Учет специфической иерархии персонала авиастроительного предприятия (ИТР/рабочие, бригады, мастера)
- Детальное отслеживание производственного цикла с перемещением изделий между участками
- Учет различных категорий изделий с их специфическими характеристиками
- Комплексный модуль испытаний с учетом лабораторий, оборудования и испытателей
- Гибкая система отчетности по всем видам запросов
- Учет движения кадров с историей изменений
- Возможность оперативного доступа для руководителей разных уровней

Ни одна из существующих систем не удовлетворяет требованиям полностью, требуется значительная доработка или создание специализированного решения.

## Цель и задачи проекта

**Целью** является проектирование и разработка информационной системы учета производства и кадров для авиастроительного предприятия.

### Основные задачи:

1. Проанализировать предметную область и бизнес-процессы авиастроительного предприятия
2. Определить требования к системе и сформировать техническое задание
3. Спроектировать архитектуру системы и модель данных
4. Разработать базу данных с учетом всех сущностей и связей
5. Разработать серверную часть (API) для работы с данными
6. Разработать клиентскую часть (веб-интерфейс) для различных категорий пользователей
7. Реализовать систему ролей и прав доступа
8. Реализовать все виды запросов согласно требованиям
9. Провести тестирование системы
10. Написать документацию для пользователей и администраторов
11. Ввести систему в эксплуатацию и провести обучение персонала

## Основные требования

### Функциональные требования

#### Управление организационной структурой
- Ведение справочника цехов предприятия
- Ведение справочника участков с привязкой к цехам
- Назначение руководителей цехов и участков

#### Управление кадрами
- Регистрация персонала с указанием категории (ИТР/рабочие)
- Учет специфических атрибутов для каждой категории персонала
- Формирование и управление бригадами
- Назначение бригадиров из числа рабочих
- Назначение мастеров на участки
- Учет движения кадров (прием, увольнение, перемещения)
- Ведение истории изменений кадрового состава

#### Управление изделиями
- Ведение справочника категорий изделий
- Регистрация изделий с учетом категории и специфических атрибутов
- Привязка изделия к цеху сборки
- Отслеживание статуса сборки изделия

#### Управление производственным процессом
- Определение циклов работ для изделий
- Назначение бригад на выполнение работ на участках
- Отслеживание перемещения изделий между участками
- Фиксация дат начала и окончания работ
- Контроль статуса выполнения работ

#### Управление испытаниями
- Ведение справочника испытательных лабораторий
- Ведение справочника оборудования лабораторий
- Регистрация испытателей с указанием специализации
- Назначение испытаний для изделий
- Фиксация используемого оборудования и испытателей
- Регистрация результатов испытаний

#### Отчетность и запросы
1. Перечень видов и количество изделий по категориям, собираемых цехом/участком/предприятием
2. Статистика собранных изделий за период по цехам/участкам
3. Отчеты по кадровому составу цеха/предприятия по категориям
4. Список участков цеха/предприятия с указанием начальников
5. Перечень работ (циклов) для конкретного изделия
6. Состав бригад участка/цеха
7. Список мастеров участка/цеха
8. Изделия, собираемые в настоящий момент участком/цехом/предприятием
9. Бригады, участвующие в сборке конкретного изделия
10. Лаборатории, проводившие испытания конкретного изделия
11. Изделия, прошедшие испытания в лаборатории за период
12. Испытатели, участвовавшие в испытаниях изделия/категории за период
13. Оборудование, использовавшееся при испытании изделия/категории за период
14. Текущая загрузка цехов/участков (изделия в сборке)

#### Дополнительные функции
- Поиск по всем сущностям системы
- Фильтрация и сортировка данных
- Экспорт отчетов в различных форматах (PDF, Excel, CSV)
- Система уведомлений о завершении этапов работ
- Журнал аудита действий пользователей

### Нефункциональные требования

#### Производительность
- Время отклика системы на запросы не более 2 секунд
- Поддержка одновременной работы до 500 пользователей
- Возможность работы с базой данных объемом до 10 млн записей

#### Безопасность
- Ролевая модель доступа (администратор, начальник цеха, начальник участка, мастер, кадровик, планировщик, оператор, аналитик)
- Разграничение прав доступа на уровне данных (руководитель видит только свое подразделение)
- Хеширование паролей пользователей (bcrypt, Argon2)
- Журналирование всех операций изменения данных
- Защита от SQL-инъекций и XSS-атак

#### Надежность
- Доступность системы 99.5% (за исключением планового обслуживания)
- Резервное копирование данных каждые 24 часа
- Возможность восстановления данных из резервной копии

#### Масштабируемость
- Модульная архитектура для возможности добавления новых функций
- Возможность масштабирования по горизонтали (несколько серверов приложений)
- Оптимизация запросов к базе данных с использованием индексов

#### Интеграция
- REST API для интеграции со сторонними системами
- Документация API в формате OpenAPI/Swagger
- Возможность импорта данных из Excel/CSV
- Экспорт данных в форматах Excel, CSV, PDF

#### Интерфейс
- Адаптивный веб-интерфейс для работы на различных устройствах
- Поддержка современных браузеров (Chrome, Firefox, Safari, Edge)
- Интуитивно понятный интерфейс с минимальным количеством кликов для выполнения операций
- Настраиваемые дашборды для разных ролей пользователей

#### Технические требования
- Использование реляционной СУБД (PostgreSQL, MySQL)
- Серверная часть на одном из языков: Python (Django/FastAPI), Java (Spring), C# (.NET)
- Клиентская часть: React/Vue.js/Angular
- Контейнеризация приложения (Docker)
- Покрытие кода тестами не менее 70%

#### Документация
- Техническая документация системы
- Пользовательская документация с инструкциями
- Документация API
- Инструкции по установке и настройке

#### Локализация
- Поддержка русского языка
- Формат дат и времени в соответствии с российскими стандартами
- Соответствие требованиям российского законодательства в части учета кадров
